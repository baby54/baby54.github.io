动态规划（Dynamic Programming）

动态规划（Dynamic Programming）

一、定义与原理

动态规划（DP）是一种通过把原问题分解为相对简单的子问题，并保存子问题的解来避免重复计算，从而高效求解最优化问题的方法。其核心思想是：

  • 最优子结构：一个问题的最优解包含其子问题的最优解。
  • 重叠子问题：递归求解时，子问题被重复计算多次，DP 通过记忆化（memoization）或自底向上填表来消除重复。

通常有两种实现方式：

  • 自顶向下（记忆化搜索）：从原问题出发递归求解，每计算一个子问题就存储结果。
  • 自底向上（表格迭代）：从最小子问题开始，按顺序逐步推出更大问题的解。

简单例子：斐波那契数列。F(n) = F(n-1) + F(n-2)，直接递归有大量重复计算；用 DP 数组保存已算出的值，时间复杂度从 O(2^n) 降至 O(n)。

二、适用场景（典型问题）

  1. 最值问题：如最长递增子序列、最小路径和、最大子数组和。
  2. 计数问题：如爬楼梯（多少种方式）、不同路径、整数拆分。
  3. 可行性问题：如能否分割等和子集、是否能到达终点。
  4. 字符串匹配/编辑距离：如最长公共子序列、编辑距离、通配符匹配。

关键词：最优、最值、方案数、是否可行、子序列、子数组、背包、区间 DP、状态机。

三、不能用的场景

  • 无最优子结构：问题的最优解不依赖于子问题的最优解，例如求有向图中的最长简单路径（子路径最优不一定能拼成全局最优）。
  • 无重叠子问题：分治法每次产生全新子问题（如快速排序），DP 没有收益。
  • 状态空间过大：状态维度爆炸（如三维以上且边界很大），时间和空间不可接受，需考虑其他算法（贪心、启发式）。
  • 存在后效性：某阶段的状态会影响后续决策的约束条件，且无法通过增加状态维度消除（例如某些路径问题需要记录访问历史）。

四、简单代码示例（Java）

问题：爬楼梯（LeetCode 70），每次可爬 1 或 2 阶，求爬到 n 阶的不同方法数。

五、📚 例题精讲与分析

📖 1. 70. 爬楼梯：斐波那契型 DP

题目描述：假设你正在爬楼梯，需要 n 阶才能到达楼顶。每次可以爬 1 或 2 个台阶，有多少种不同的方法？

核心转化：到达第 i 阶的方法数 = 到达 i-1 阶的方法数 + 到达 i-2 阶的方法数（因为最后一步跨 1 阶或 2 阶）。

算法步骤：

  1. 定义 dp[i] 为到达第 i 阶的方法数。
  2. 初始化 dp[1]=1，dp[2]=2。
  3. 从 i=3 到 n，执行 dp[i] = dp[i-1] + dp[i-2]。
  4. 返回 dp[n]。

复杂度分析：时间 O(n)，空间 O(n)；可优化至 O(1)。

易错点 / 小技巧：n=0 时题意通常不出现；注意边界 n=1 直接返回。

📖 2. 53. 最大子数组和：经典 Kadane 算法

题目描述：给定整数数组 nums，找出一个具有最大和的连续子数组（子数组最少包含一个元素），返回其最大和。

核心转化：dp[i] 表示以 nums[i] 结尾的最大子数组和。状态转移：dp[i] = max(nums[i], dp[i-1] + nums[i])。最终答案是 max(dp[0..n-1])。

算法步骤：

  1. 初始化 dp[0] = nums[0]，maxSum = nums[0]。
  2. 遍历 i 从 1 到 n-1：
    • dp[i] = Math.max(nums[i], dp[i-1] + nums[i])
    • 更新 maxSum = Math.max(maxSum, dp[i])
  3. 返回 maxSum。

Java 代码：

复杂度分析：时间 O(n)，空间 O(1)。

易错点 / 小技巧：不能直接取 max(dp[i-1]+nums[i], dp[i-1])，因为子数组必须连续；如果全为负数，答案是最大负数而非 0。

📖 3. 300. 最长递增子序列：O(n²) 与 O(n log n)

题目描述：给你一个整数数组 nums，找到其中最长严格递增子序列的长度（子序列不要求连续）。

核心转化：定义 dp[i] 为以 nums[i] 结尾的最长递增子序列长度。转移：dp[i] = max(dp[j]) + 1，其中 j < i 且 nums[j] < nums[i]。最终答案为 max(dp)。

算法步骤（O(n²)）：

  1. 初始化 dp 全为 1。
  2. 双重循环：外层 i，内层 j (0..i-1)，若 nums[j] < nums[i] 则 dp[i] = max(dp[i], dp[j]+1)。
  3. 返回 max(dp)。

易错点 / 小技巧：子序列不要求连续，所以内层循环 j 从 0 到 i-1 全检查；O(n log n) 方案中维护一个 tails 数组，表示长度为 k 的递增子序列的最小末尾值。

📖 4. 1143. 最长公共子序列：二维 DP

题目描述：给定两个字符串 text1 和 text2，返回它们的最长公共子序列的长度（子序列不要求连续）。

核心转化：定义 dp[i][j] 表示 text1[0..i-1] 和 text2[0..j-1] 的 LCS 长度。转移：

  • 若 text1[i-1] == text2[j-1]，则 dp[i][j] = dp[i-1][j-1] + 1
  • 否则 dp[i][j] = max(dp[i-1][j], dp[i][j-1])

算法步骤：

  1. 创建 (m+1)*(n+1) 的 dp 数组，初始化为 0。
  2. 双重循环 i=1..m，j=1..n，按转移公式填表。
  3. 返回 dp[m][n]。

复杂度分析：时间 O(m×n)，空间 O(m×n)，可优化至 O(min(m,n))。

易错点 / 小技巧：dp 数组多开一行一列避免了边界判断；字符相等时必须从 [i-1][j-1] 转移，不能从上边或左边，否则会重复计数。

📖 5. 322. 零钱兑换：完全背包问题

题目描述：给定不同面额的硬币 coins 和一个总金额 amount，计算可以凑成总金额所需的最少硬币个数。如果无法凑成，返回 -1。每种硬币数量无限。

核心转化：完全背包（每种物品无限取），求恰好装满背包的最小价值（硬币个数）。定义 dp[i] 为凑成金额 i 所需的最少硬币数。初始化 dp[0]=0，其余为 amount+1（不可达标记）。转移：dp[i] = min(dp[i], dp[i - coin] + 1)。

算法步骤：

  1. 创建 dp 数组长度 amount+1，填充大值 amount+1。
  2. dp[0] = 0。
  3. 外层遍历金额 i 从 1 到 amount，内层遍历每种硬币 coin，若 i >= coin，更新 dp[i] = Math.min(dp[i], dp[i-coin] + 1)。
  4. 若 dp[amount] == amount+1 返回 -1，否则返回 dp[amount]。

Java 代码：

复杂度分析：时间 O(amount × n)，空间 O(amount)，n 为硬币种类数。

易错点 / 小技巧：初始化不能用 Integer.MAX_VALUE，否则加 1 会溢出；外层遍历金额，内层遍历硬币（完全背包常见顺序）；如果先硬币后金额且用正序遍历，也是完全背包，结果相同。

📖 6. 72. 编辑距离：困难题，二维 DP

题目描述：给你两个单词 word1 和 word2，请返回将 word1 转换成 word2 所使用的最少操作数。你可以对一个单词进行如下三种操作：插入一个字符、删除一个字符、替换一个字符。

核心转化：定义 dp[i][j] 表示 word1 的前 i 个字符转换成 word2 的前 j 个字符所需的最少操作数。转移：

  • 若 word1[i-1] == word2[j-1]，则 dp[i][j] = dp[i-1][j-1]（无需操作）
  • 否则 dp[i][j] = 1 + min(删除：dp[i-1][j], 插入：dp[i][j-1], 替换：dp[i-1][j-1])

算法步骤：

  1. 初始化边界：dp[i][0] = i（全部删除），dp[0][j] = j（全部插入）。
  2. 双重循环 i=1..m，j=1..n，按转移公式填表。
  3. 返回 dp[m][n]。

易错点 / 小技巧：注意三种操作的对应关系：删除对应 dp[i-1][j]，插入对应 dp[i][j-1]，替换对应 dp[i-1][j-1]；初始化边界容易遗漏。
