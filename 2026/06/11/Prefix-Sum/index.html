前缀和算法（Prefix Sum）

想练习前缀和算法，不妨从下面这几道思路清晰、特征明显的力扣题目开始。我按照从基础到进阶的顺序帮你整理好了，每道题都刚好对应前缀和算法的一种经典应用场景。

📚 例题精讲与分析

📖 1. 303. 区域和检索 - 数组不可变：前缀和的“基本功”

这道题是学习前缀和的最佳入门题。它会让你的 NumArray 类支持 sumRange(i, j) 方法，这个方法需要被高效地调用很多次。这里的核心优化思路，就是利用一维前缀和将每次区间求和的时间复杂度从 O(n) 降到 O(1)。

你可以这样理解：预先计算一个新数组 preSum，preSum[i] 存储原数组 nums 前 i 个元素的和。之后，nums 从 i 到 j 的区间和就等于 preSum[j+1] - preSum[i]，是不是很方便？。

📖 2. 724. 寻找数组的中心下标 (Find Pivot Index)：用左右前缀和来“找平衡”

这道题要求找到一个中心下标，使得它左侧所有元素之和等于右侧所有元素之和。

它的解法非常优雅：你可以利用一次遍历，并借助前缀和数组和后缀和数组来求解。在遍历时，preSum[i] 用于表示索引 i 左侧元素的和，sufSum[i] 表示它右侧元素的和。如果某一位置上左右两个和相等，那这个位置就是答案。

📖 3. 560. 和为 K 的子数组 (Subarray Sum Equals K)：搭配哈希表的“高效检索”

这道题的进阶之处在于，你不仅需要快速求区间和，还需要高效地“查找”。问题是要在数组中找到所有和为 K 的连续子数组个数。

单纯的暴力解法是 O(n^2)，但配合一个哈希表就能优化到 O(n)。
算法的核心思路是用一个哈希表来存储已经出现过的“前缀和”及其出现的次数。当我们遍历数组并累加当前前缀和 curSum 时，只要去哈希表中查找有多少个之前的前缀和等于 curSum - K，就能在 O(1) 时间内找到所有符合条件的子数组。

📖 4. 525. 连续数组 (Contiguous Array)：巧妙的“转化”思想

这道题要求找出含有相同数量 0 和 1 的最长连续子数组。

它用到了一种巧妙的“转化”技巧：将数组中的所有 0 替换为 -1。这样一来，问题就变成了寻找元素和为 0 的最长连续子数组。下一步，我们又可以用到哈希表来辅助：用一个哈希表记录每个前缀和第一次出现的位置。当同一个前缀和再次出现时，就意味着这两次出现之间的子数组元素和为 0，也就找到了一个符合要求的子数组，并可以很方便地计算出它的长度。

📖 5. 1109. 航班预订统计 (Corporate Flight Bookings)：反向运用“差分数组”

这道题很适合运用差分数组，你可以把它看作是前缀和的一种逆向应用。

题目是：有一个长度为 n 的航班数组，以及一系列预订记录，每条记录 (i, j, k) 表示从第 i 站到第 j 站的航班每趟都多坐了 k 个人。要求返回最终每个航班上的人数。

对于这种对数组的一个连续区间进行统一增减的操作，使用差分数组是最高效的。对差分数组求一次前缀和，就能复原出原数组每个位置的最终结果。
