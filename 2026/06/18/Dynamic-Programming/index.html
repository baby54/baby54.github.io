动态规划（Dynamic Programming)

一、定义与原理

动态规划（Dynamic Programming，简称 DP） 是一种通过将原问题分解为相互重叠的子问题，并存储子问题的解以避免重复计算，从而高效求解最优化问题的算法思想。

其核心逻辑可以概括为三个关键词：

  1. 最优子结构（Optimal Substructure）
    一个问题的最优解包含其子问题的最优解。即，可以通过组合子问题的最优解来构造原问题的最优解。

  2. 重叠子问题（Overlapping Subproblems）
    递归求解时，子问题会被重复计算多次。DP 通过**记忆化（Memoization）或自底向上填表（Tabulation）**来复用计算结果。

  3. 状态转移方程（State Transition Equation）
    描述子问题之间如何递推的数学关系，是 DP 的核心。

简单例子：爬楼梯问题（70. 爬楼梯）。
要爬到第 n 级，可以从第 n-1 级跨 1 步，或从第 n-2 级跨 2 步。
所以 dp[n] = dp[n-1] + dp[n-2]，这就是状态转移方程。
初始条件：dp[1]=1, dp[2]=2。通过自底向上计算即可得到结果。

二、适用场景（典型问题）

动态规划最擅长解决具有以下特征的问题：

  1. 最优化问题（求最大值、最小值、最少/最多数量）

    • 关键词：最小路径和、最大子数组、最长递增子序列、最小编辑距离。

  2. 计数问题（求方案数、组合数）

    • 关键词：不同路径、解码方法、不同的二叉搜索树、零钱兑换 II。

  3. 可行性问题（判断能否达到目标）

    • 关键词：分割等和子集、单词拆分、能否到达终点。

  4. 具有明确阶段和状态转移的序列/网格问题

    • 关键词：股票买卖（多阶段决策）、区间合并（戳气球）、树形依赖（打家劫舍 III）。

三、不能用的场景

动态规划并非万能，以下情况不适用或效率极低：

  1. 存在后效性（State Aftereffect）
    即当前决策会影响后续状态的独立性，子问题的解无法独立确定。
    反例：带有环的最短路径（需用 Bellman-Ford 或 Floyd，而非简单 DP）。

  2. 子问题不重叠（分治更优）
    若递归树没有重复子问题，DP 的存储开销无意义，用分治（如快速排序）即可。

  3. 状态空间过大，无法有效压缩或存储
    例如，状态维度极高（如多维背包）、状态数量指数级且无法剪枝时，DP 会超时/超内存。

  4. 问题本身是 NP-hard 且无伪多项式解法
    例如，一般的旅行商问题（TSP）用状态压缩 DP 也只能处理小规模（n ≤ 20）。
